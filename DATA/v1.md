# GIMB BUSINESS — V1 DATA SPEC

### (Trade & Lead Operating System)

---

## 1. Purpose of V1

**V1 exists to answer one question perfectly:**

> "I need something for my trade — who can help me right now?"

V1 transforms GIMB from a **directory** into a **deal flow engine**.

---

## 2. Prerequisites for V1

V1 only launches after V0 achieves:

* ≥ 500 verified businesses
* ≥ 100 CHAs profiled
* ≥ 50 CHA ratings submitted
* Active daily usage pattern established

Without V0 trust, V1 transactions won't happen.

---

## 3. Geographic Scope

Same as V0: **Kutch District**

**Primary Focus:**
* Gandhidham (GIMB)
* Kandla Port

**Secondary:**
* Adipur, Bhuj, Mundra, Anjar, Other Kutch

No expansion outside Kutch until V1 is proven.

---

## 4. Core Data Objects (V1)

### 4.1 RFQ (Request for Quotation)

**Purpose:** Structured way to post trade requirements.

**Fields:**

* rfq_id (UUID)
* requester_business_id (FK)
* title
* rfq_type
  *(Service Request, Product Sourcing, Logistics Need, CHA Requirement, Consultation)*
* category
  *(Import, Export, Warehousing, Transport, Customs, Documentation)*
* description (detailed requirement)
* quantity (if applicable)
* unit
* budget_range (optional)
* delivery_timeline
* preferred_locality (Gandhidham / Kandla / Adipur / Bhuj / Mundra / Anjar / Any Kutch)
* attachments (array: specs, images, documents)
* visibility
  *(public / invited only / category-specific)*
* status
  *(open / in-negotiation / closed / expired)*
* expires_at
* created_at
* updated_at

---

### 4.2 Quote (Response to RFQ)

**Purpose:** Structured bid/offer against an RFQ.

**Fields:**

* quote_id (UUID)
* rfq_id (FK)
* responder_business_id (FK)
* quoted_price
* price_breakdown (optional, JSON)
* validity_days
* delivery_commitment
* terms_conditions
* attachments (array)
* status
  *(submitted / viewed / shortlisted / accepted / rejected / withdrawn)*
* submitted_at
* updated_at

---

### 4.3 Lead Notification

**Purpose:** Alert businesses about relevant opportunities.

**Fields:**

* notification_id (UUID)
* business_id (FK)
* rfq_id (FK)
* match_reason
  *(category match / service match / location match)*
* match_score (internal, 0–100)
* notification_status
  *(pending / sent / opened / acted)*
* sent_at
* opened_at

---

### 4.4 Deal Record

**Purpose:** Track completed transactions for trust building.

**Fields:**

* deal_id (UUID)
* rfq_id (FK)
* quote_id (FK)
* buyer_business_id (FK)
* seller_business_id (FK)
* deal_value (optional, self-reported)
* deal_status
  *(initiated / in-progress / completed / disputed)*
* completion_date
* feedback_submitted (boolean)
* created_at

---

## 5. Enhanced CHA Rating System (V1)

### 5.1 CHA Performance Metrics

**Purpose:** Move from basic ratings to structured performance tracking.

**Additional Fields (extends V0):**

* business_id (FK)
* total_ratings_count
* avg_timeliness_score
* avg_documentation_score
* avg_communication_score
* overall_score (weighted average)
* response_rate (% of enquiries responded)
* avg_response_time
* deals_completed_via_gimb
* last_rating_received_at

---

### 5.2 Detailed CHA Review

**Purpose:** Rich feedback beyond star ratings.

**Fields:**

* review_id (UUID)
* cha_business_id (FK)
* reviewer_business_id (FK)
* deal_id (FK, optional — links to actual transaction)
* cargo_type_handled
* shipment_value_range
  *(< ₹1L / ₹1L–10L / ₹10L–50L / ₹50L+ )*
* rating_dimensions:
  * timeliness (1–5)
  * documentation_accuracy (1–5)
  * communication (1–5)
  * problem_resolution (1–5) ← NEW
  * value_for_money (1–5) ← NEW
* detailed_feedback (mandatory, 50+ words)
* would_recommend (yes / no / maybe)
* verified_transaction (boolean)
* created_at
* moderation_status

## 6. Subscription System (V1)

### 6.1 Subscription Plan

**Purpose:** Define available subscription tiers.

**Fields:**

* plan_id (UUID)
* plan_name
  *(Starter / Professional / Enterprise)*
* plan_type
  *(monthly / quarterly / annual)*
* price_inr
* features (JSON array):
  * listing_visibility_boost
  * rfq_access_limit (per month)
  * quote_limit (per month)
  * lead_notifications (yes/no)
  * priority_support (yes/no)
  * featured_placement (yes/no)
  * analytics_access (basic/advanced)
* active (boolean)
* created_at

---

### 6.2 Business Subscription

**Purpose:** Track which businesses have subscribed.

**Fields:**

* subscription_id (UUID)
* business_id (FK)
* plan_id (FK)
* status
  *(active / expired / cancelled / grace_period)*
* started_at
* expires_at
* auto_renew (boolean)
* payment_method
* last_payment_at
* next_billing_at

---

## 7. Messaging System (V1)

### 7.1 Conversation

**Purpose:** Contextual communication around RFQs.

**Fields:**

* conversation_id (UUID)
* rfq_id (FK, optional)
* participants (array of business_ids)
* conversation_type
  *(rfq_discussion / general_enquiry / support)*
* status
  *(active / archived / reported)*
* created_at
* last_message_at

---

### 7.2 Message

**Purpose:** Individual messages within conversations.

**Fields:**

* message_id (UUID)
* conversation_id (FK)
* sender_business_id (FK)
* content (text)
* attachments (array)
* read_status (boolean)
* sent_at

⚠️ **V1 Messaging Rules:**
* No spam — messages must be contextual
* Business-to-business only
* Reportable for abuse
* No cold outreach without RFQ context

## 9. User Roles (V1 Extensions)

### Existing Roles (from V0):
* Admin
* Verified Business User
* Public Viewer

### New V1 Roles:

* **Subscriber (Paid)**
  * Full RFQ access
  * Can submit quotes
  * Lead notifications
  * Analytics dashboard

* **RFQ Poster**
  * Any verified business can post RFQs
  * Free to post, paid to respond

---

## 10. V1 Success Metrics

V1 is considered successful when:

* ≥ 200 RFQs posted per month
* ≥ 500 quotes submitted per month
* ≥ 50 deals closed via GIMB
* ≥ 100 paying subscribers
* ≥ 30% RFQ-to-quote conversion rate
* Businesses say: "I found them on GIMB"

---

## 11. What V1 Explicitly Does NOT Include

To avoid scope creep:

* Payments processing (comes in V2)
* Resource sharing / rentals (comes in V2)
* Credit scoring (comes in V2)
* AI-powered matching (future)
* Multi-city expansion (after V1 proven)
